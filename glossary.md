# Glossary

Definizioni univoche per evitare ambiguità con cliente e team.

| Termine | Definizione operativa |
|---------|----------------------|
| **PED** | Piano Editoriale: insieme dei post di un mese su uno o più canali social, con copy, formato, asset, calendario di pubblicazione |
| **Post** | Singola unità di contenuto pubblicata su un canale social (carosello, reel, statico, story) |
| **Carosello** | Post multi-slide (tipicamente Instagram/LinkedIn) |
| **Campagna** | Investimento pubblicitario a pagamento con obiettivo, KPI, audience, budget, creatività |
| **KPI** | Key Performance Indicator: metrica concordata in contratto come target |
| **Conversione** | Azione di valore tracciata (acquisto, lead, contatto) |
| **Strategia** | Documento annuale che definisce target, pillar, formati, calendario macro, KPI |
| **Pillar** | Macro-tema di contenuto su cui si articola la strategia |
| **Touch-point email** | Mail di passaggio fase che formalizza decisioni e richieste di approvazione |
| **Gap-check** | Verifica preventiva di completezza degli input prima di produrre un output |
| **MVI** | Mitigazione esplicita del rischio per cliente nuovo senza dati storici |
| **Silenzio-assenso** | Modalità di approvazione: se nessuna obiezione entro deadline, si procede |
| **Slug** | Identificatore cliente in kebab-case (es. `muvi-neogela`) |
| **Cartella periodo** | `YYYY-MM` per ped/campaigns, `YYYY-MM-DD` per report, `YYYY` per strategy |
| **Approvato** | Stato di un artefatto che ha ricevuto OK esplicito o silenzio-assenso scaduto |
| **Feedback** | Commento ricevuto dal cliente o team durante validazione |
| **Evergreen** | Contenuto non datato, riutilizzabile (vive in `context/posts-library/`) |
| **Pilota** | Primo cliente migrato sul framework, usato come reference |
